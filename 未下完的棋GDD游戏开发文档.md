# 《未下完的棋》（暂定）游戏开发文档

> 最后更新：2026年5月10日

---

## 1. 游戏概述

### 1.1 基本信息

- **游戏名称**：未下完的棋（暂定）
- **游戏类型**：Roguelike · 围棋变体 · 策略计分
- **目标平台**：PC / 移动端
- **核心灵感**：围棋 + Balatro（小丑牌）+ Gambonanza

### 1.2 核心概念

玩家执黑与电脑白棋在小棋盘对弈。棋子数量受限，用尽则终局。黑棋通过吃子触发斜向染色积累分数价值，终局计算围眼总分。过关后进入商店购买道具强化，进入下一关。

循环：**“对局 → 计分 → 商店 → 变强 → 下一关 / Boss”**。

## 2. 核心规则

### 2.1 围棋基础（保留）

- 玩家执黑，电脑执白，黑先
- 标准围棋规则：落子、气、提子、禁着
- 双方各有固定棋子数量，落子消耗，用尽则只能虚手
- 虚手无连续限制
- 终局条件：双方棋子数均为 0 **且** 连续两次虚手，对局结束
- 特殊：终局不清理死子，盘上所有棋子均视为活棋

### 2.2 染色机制

| 项目 | 说明 |
| --- | --- |
| 触发条件 | 仅黑棋吃白棋时触发（难度 0 单向） |
| 作用范围 | 被吃白子所在点的四个斜对角棋格 |
| 效果 | 斜对角棋格分数价值永久 +1 |
| 叠加 | 无上限，可重复染色 |
| 设计意图 | 与围棋正交规则分离，创造独立的分数积累层 |

**图示（4×4 棋盘）**：若白子位于 (2,2) 被吃，则染色格为 (1,1) (1,3) (3,1) (3,3)。

### 2.3 终局计分

**眼的定义**：终局时，被黑棋棋子完全包围的空交叉点即为“眼”。

**计分时机**：仅在终局时一次性计算。局中被破坏的眼不影响已有染色，但也不得分。

**单眼得分公式**：

```
单眼得分 = 1（基础分） + 该点有效斜对角格分数价值之和
```

- 中腹眼：加 4 个斜对角格分数
- 边线眼：加 2 个斜对角格分数
- 角部眼：加 1 个斜对角格分数

**过关条件**：所有眼得分累加 ≥ 关卡目标分。

## 4. 不对称对抗

### 4.1 玩家（黑棋）

- 可查看白棋剩余棋子数
- 唯一拥有染色能力（吃白子触发）
- 拥有商店道具、药水、特殊棋子
- 剩余未用棋子可转化为货币 / 分数加成

### 4.2 电脑（白棋）

- 基本策略（难度 0）：
  1. 优先破坏黑棋潜在眼位
  2. 侵入黑棋势力范围，迫使黑棋消耗棋子
  3. 保护自身不被吃（因被吃会染色）
- 行为树随关卡和 Boss 战逐步复杂化

### 4.3 棋子数量博弈

- 黑棋子多：白棋可能无子可下，黑棋可连续虚手推进终局
- 白棋子多：白棋可下入黑棋领地破坏眼位

## 5. 商店与成长系统

过关后进入商店，可购买以下四类内容。

### 5.1 药水（一次性消耗）

| 名称 | 效果 |
| --- | --- |
| 飞刀 | 连下两手 |
| 炸弹 | 下一颗棋子到对方棋子上，改变颜色 |
| 原子弹 | 移除 2×2 区域所有棋子 |
| 染色喷雾 | 指定棋格分数价值 +1 |
| 悔棋 | 撤回最近一步 |

### 5.2 道具（常驻被动）（暂定）

| 类型 | 示例效果 |
| --- | --- |
| 底分+ | 每个眼基础分 +1 |
| 倍率 | 中腹眼 ×1.5，边角眼 ×2 |
| 乘倍率 | 总得分 ×（1 + 0.2 × 染色格数） |
| 运转 | 每吃一子抽取道具牌 |
| 经济 | 剩余棋子价值翻倍，过关额外金钱 |

### 5.3 特殊棋子

| 名称 | 效果 |
| --- | --- |
| 发光棋子 | 周围正交四交叉点若成眼，基础分 ×2 |
| 染色核心子 | 落子时相邻格染色 +1 |
| 种子子 | 落子时相邻格染色 +1 |

### 5.4 机巧道具（暂定）

- 棋子上限增加（如每局 +2）
- 基础染色值提升（染色时额外 +1）
- 起始金钱增加

## 6. 难度系统

### 6.2 难度扩展接口

| 难度 | 变化 |
| --- | --- |
| 难度 0 | 单向染色（仅黑吃白触发） |
| 难度 1 | 双向染色，或白棋获得染色护盾 |
| 难度 N | 白棋棋子增加，染色衰减机制等 |

## 附录：关键术语

| 术语 | 定义 |
| --- | --- |
| 染色 | 棋盘格分数价值永久提升，由黑吃白触发 |
| 眼 | 终局时被黑棋围住的空交叉点，计分基本单位 |
| 斜对角 | 坐标 (x±1, y±1) 的四个棋格，染色作用范围 |
| 虚手 | 放弃落子，不消耗棋子 |
| 棋子余量 | 双方剩余可落子数 |
